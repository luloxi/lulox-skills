Live docs first

Use current Midnight docs, hello-world guides, and MCP posts before writing code so the workflow stays current.

2. Architecture

Onchain/offchain split

3. Product

Polished by default

Workflow

A simple loop that keeps improving

1
Clarify the project goal and whether it needs frontend, backend, contracts, or all three.
2
Pull current Midnight docs and any relevant MCP references before coding.
3
Scaffold the repo with contracts, apps, packages, docs, and tests.
4
Ship the smallest working version, verify it, then patch the skill with what was learned.
